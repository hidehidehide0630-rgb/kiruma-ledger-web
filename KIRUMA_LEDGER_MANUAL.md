## 1. はじめに
### 🌟 コンセプト：ライフログを経理へ
KIRUMA Ledgerは、「記帳のために作業をする」という苦痛を排除します。日々の食事管理や買い物の記録（ライフログ）が、自動的に確定申告用の仕訳（経理）に変わります。

## 2. アプリの起動方法
日常的にアプリを使用するための起動手順です。

### 🚀 開発サーバーの立ち上げ
1. ターミナルを開き、会社のルートディレクトリから適切な場所に移動します。
   ```powershell
   cd ".agent\skills\hq-tax-affairs\tax-affairs-for-blue-return"
   ```
2. 以下のコマンドを実行してサーバーを起動します。
   ```powershell
   npm run dev
   ```
3. 起動後、ブラウザで [http://localhost:3000](http://localhost:3000) にアクセスします。

### 🛠️ 初回・変更時のセットアップ
データベースのスキーマを変更した場合や、初期状態から開始する場合は以下を実行してください。
- **クライアント生成**: `npx prisma generate`
- **DBシード（初期データ投入）**: `npx prisma db seed`

## 2. 家計・献立マネージャー
日常の食生活と予算を最適化する機能です。

### 📅 献立の自動生成 (`/household/setup`)
AIと独自のレシピデータベースを組み合わせ、期間と予算に合わせた最適な献立を提案します。
- **期間設定**: 3日間〜14日間で柔軟に設定可能。
- **予算逆算ロジック**: 月間予算の残りから「今回の買い物」で使える金額を自動算出し、予算内に収まるレシピを組み合わせます（5%の予備費を自動確保）。
- **パントリー活用**: 冷蔵庫にある余り食材（パントリー）を入力することで、それを優先的に使い切るレシピが選ばれます。

### 🔥 バイタリティモード (Vitality Mode)
男性機能の維持・向上と血管の健康に特化した栄養学的アルゴリズムです。
- **重点食材**: 亜鉛（牡蠣、赤身肉）、アルギニン・シトルリン（鶏胸肉、エビ、スイカ）、アリシン（ニラ、ニンニク）を豊富に含むレシピが優先されます。
- **AI分析**: 生成された献立ごとに、栄養学的な「バイタリティポイント」が解説されます。

## 3. Smart Sync（経理同期）
本アプリ最大の特徴である、家計簿とビジネス仕訳の連携機能です。

### 💸 支出の登録 (`/household/entry`)
スーパーでの買い物や日常の支出を記録する際、**「事業経費として計上」**トグルを有効にできます。

#### 同期の仕組み：
1. **家計簿**: `支出` として全額が記録されます。
2. **ビジネス仕訳**: 同時に以下の複式簿記仕訳が自動生成されます。
   - **借方 (Debit)**: ユーザーが選択した費用科目（例：接待交際費、消耗品費）
   - **貸方 (Credit)**: **事業主借**（個人の財布から事業のために支払ったことを意味します）

> [!TIP]
> **按分について**: 通信費や光熱費など、家事按分が必要な科目は「家事按分設定」で予め割合を決めておくと、決算時に一括で経費参入額が計算されます。

## 4. 本格的な会計管理
確定申告に必要な帳票作成や取引の管理を行います。

### 📊 取引一覧 (`/transactions`)
自動同期された取引や手動入力された仕訳を一覧で確認できます。
- **論理削除**: 電子帳簿保存法に対応するため、削除されたデータも履歴としてシステム内に保持されます。
- **編集**: 同期された取引の内容（摘要や金額）を修正すると、対応する家計簿データも整合性が保たれます。

### 📈 各種帳票 (`/reports`)
- **損益計算書 (P/L)**: 収益・費用の状況をリアルタイムに集計。
- **貸借対照表 (B/S)**: 資産と負債のバランスを可視化。
- **確定申告ナビ**: 青色申告決算書作成までのステップをガイドします。
